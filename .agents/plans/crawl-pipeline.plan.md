# Plan: Property Crawl Pipeline

## Summary

Replace the MarketResearcher's "ask Claude to guess listings from web search snippets" approach with a real crawling pipeline that uses Firecrawl to scrape actual rental listing pages from Zillow, Apartments.com, and Rent.com. The pipeline extracts structured data from real HTML, normalizes it into the existing `properties` schema, deduplicates across sources, and persists verified listings. The MarketResearcher agent shifts from web_search to querying real crawled data, with on-demand crawls when results are insufficient.

## User Story

As a **renter searching for properties in LA**
I want to **see real, verified listings from actual rental websites**
So that **I can trust the prices, addresses, and details are accurate and up-to-date**

- **Action**: CREATE (via Supabase MCP `apply_migration`)

  ```sql
  -- Add crawl tracking columns to properties table
  ALTER TABLE public.properties ADD COLUMN IF NOT EXISTS source_id TEXT;
  ALTER TABLE public.properties ADD COLUMN IF NOT EXISTS source_name TEXT;
  ALTER TABLE public.properties ADD COLUMN IF NOT EXISTS source_url TEXT;
  ALTER TABLE public.properties ADD COLUMN IF NOT EXISTS source_urls TEXT[] DEFAULT '{}';
  ALTER TABLE public.properties ADD COLUMN IF NOT EXISTS last_crawled_at TIMESTAMPTZ;
  ALTER TABLE public.properties ADD COLUMN IF NOT EXISTS last_verified_at TIMESTAMPTZ;
  ALTER TABLE public.properties ADD COLUMN IF NOT EXISTS is_active BOOLEAN DEFAULT true;
  ALTER TABLE public.properties ADD COLUMN IF NOT EXISTS raw_data JSONB;

  -- Index for active listing queries (most common query path)
  CREATE INDEX IF NOT EXISTS idx_properties_active ON public.properties (is_active) WHERE is_active = true;
  -- Index for source-based dedup lookups
  CREATE INDEX IF NOT EXISTS idx_properties_source ON public.properties (source_name, source_id);
  -- Index for address-based dedup
  CREATE INDEX IF NOT EXISTS idx_properties_address ON public.properties (address);

  -- Crawl run tracking
  CREATE TABLE IF NOT EXISTS public.crawl_runs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    source_name TEXT NOT NULL,
    search_params JSONB NOT NULL,
    started_at TIMESTAMPTZ DEFAULT now(),
    completed_at TIMESTAMPTZ,
    status TEXT DEFAULT 'running' CHECK (status IN ('running', 'completed', 'failed')),
    listings_found INTEGER DEFAULT 0,
    listings_new INTEGER DEFAULT 0,
    listings_updated INTEGER DEFAULT 0,
    listings_deactivated INTEGER DEFAULT 0,
    error_message TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
  );

  -- Per-listing source tracking (one property can appear on multiple sites)
  CREATE TABLE IF NOT EXISTS public.crawl_sources (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    property_id UUID NOT NULL REFERENCES public.properties(id) ON DELETE CASCADE,
    source_name TEXT NOT NULL,
    source_id TEXT,
    source_url TEXT NOT NULL,
    last_seen_at TIMESTAMPTZ DEFAULT now(),
    price_at_source NUMERIC,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT now(),
    UNIQUE(property_id, source_name)
  );

  -- RLS for crawl_runs (service role only — no user access needed)
  ALTER TABLE public.crawl_runs ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "Service role full access on crawl_runs"
    ON public.crawl_runs FOR ALL
    USING (auth.role() = 'service_role');

  -- RLS for crawl_sources (users can read, service role can write)
  ALTER TABLE public.crawl_sources ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "Authenticated users can read crawl_sources"
    ON public.crawl_sources FOR SELECT
    USING (auth.role() = 'authenticated');
  CREATE POLICY "Service role full access on crawl_sources"
    ON public.crawl_sources FOR ALL
    USING (auth.role() = 'service_role');
  ```

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Firecrawl markdown output varies by site | Start with Zillow, iterate on extraction. Use LLM-assisted extraction as fallback (pass markdown to Claude with structured output prompt). |
| Rate limiting / IP blocking by rental sites | Conservative defaults (5 req/min per source), Firecrawl handles proxy rotation. Browserbase as fallback. |
| Extraction logic breaks when sites update HTML | Each adapter isolated. Log raw markdown to `raw_data` JSONB column for debugging. |
| Database types out of sync after migration | Regenerate types immediately after migration (Task 11). |
| Edge Function can't import Node.js modules | Edge Function calls Next.js API route via HTTP instead of importing crawl modules directly. |
| Empty or low-quality crawl results initially | Keep the web_search Claude approach as a deprecated fallback in MarketResearcher, gated behind a feature flag or result count check. |
| Dedup false positives merge different listings | Start with high-confidence matches only (exact address). Add fuzzy matching incrementally. |
